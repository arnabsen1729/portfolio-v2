Consistent hashing was introduced to solve a fairly narrow problem: how do you
distribute cache load across a growing set of web caches without reshuffling
almost everything every time you add a machine<Sidenote>Karger et al.,
"Consistent Hashing and Random Trees," STOC 1997 — originally framed around
Akamai's caching layer.</Sidenote>. Naive modulo hashing (`hash(key) % N`)
falls apart the moment `N` changes, because nearly every key maps to a
different bucket.

## The ring

The trick is to hash both the keys and the nodes onto the same circular
keyspace, then assign each key to the next node clockwise from it on the
ring.

```python
import bisect
import hashlib

class ConsistentHashRing:
    def __init__(self, nodes=(), vnodes=150):
        self.vnodes = vnodes
        self.ring = {}
        self.sorted_keys = []
        for node in nodes:
            self.add_node(node)

    def _hash(self, key: str) -> int:
        return int(hashlib.sha1(key.encode()).hexdigest(), 16)

    def add_node(self, node: str) -> None:
        for i in range(self.vnodes):
            h = self._hash(f"{node}#{i}")
            self.ring[h] = node
            bisect.insort(self.sorted_keys, h)

    def get_node(self, key: str) -> str:
        h = self._hash(key)
        idx = bisect.bisect(self.sorted_keys, h) % len(self.sorted_keys)
        return self.ring[self.sorted_keys[idx]]
```

Adding or removing a node only remaps the keys that fall in the affected arc
of the ring — on average `K/N` keys move, not all of them.

## Virtual nodes fix the load-balancing problem

A raw ring with one point per physical node produces very uneven arc
lengths<Sidenote>With only a handful of nodes, the gaps between them on the
ring can vary by an order of magnitude — some nodes get almost no traffic,
others get flooded.</Sidenote>. The fix, used in the snippet above, is to hash
each node to many points on the ring (`vnodes`), which averages out the
variance in arc length and therefore load.

This is the same idea underneath DynamoDB's partitioning, Cassandra's token
ranges, and most CDN request routing — a genuinely old idea that just never
stopped being the right answer.
